## Evidence Rules

- `bounded` means the current slice has behavior, record, artifact, diagnostic,
  or public-boundary evidence in checked-in tests.
- Parser-only, type-check-only, descriptor-only, or scaffold-only evidence is
  not executable production support.
- Production gates must be proved by public behavior, GPU-owned records,
  persisted artifacts, stable diagnostics, or measured local artifacts. Tests
  that read compiler/shader source or grep implementation text are not
  readiness evidence.
- The CPU may load files, schedule work, and write artifacts. It must not parse,
  rewrite, type-check, monomorphize, link, or patch program semantics as a
  production fallback.

## Current Pass Audit Notes

- The measurement scaffold now records the paper pass order from the checked-in
  GPU compilation papers separately from local Lanius evidence. The local
  scaffold can cite that order only as a baseline architecture contract; it
  cannot promote paper timing tables, the narrow current optimization-stage
  coverage, or bounded/fail-closed pass contracts into performance or scaling
  claims.
- Type checking has moved visible declarations, predicate obligation arguments,
  and method contract names/owners onto parser-owned HIR records rather than
  late consumer-side source discovery. Struct member and
  struct-literal field lookup now consumes parser/type metadata before the
  consumers run: the type-instance pass seeds/sorts/scatters a field-key table
  keyed by `(struct_node, field_name_id)`, parser passes publish nearest
  struct-literal statement context rows, and type checking maps those into
  literal-keyed context rows before field typing performs range queries. It
  still has bounded obligation windows, parser-local record derivation, and
  incomplete nested contextual type propagation for struct literals.
  Those are alpha/fail-closed limits, not production scalability evidence.
- Source-pack package/import/link control-plane records now enforce coarse
  library ordering plus canonical persisted ids/ranges for library
  dependencies, package-lock import graph dependencies and edges, job
  dependencies, job-batch dependencies, work-queue edges, artifact ranges, and
  link-batch inputs. This is still scheduling and replay integrity, not semantic
  module resolution or real GPU linking.
- x86 lowering still has bounded record consumers that are not final
  paper-style joins: `x86_return_match_records` now materializes the direct
  return-statement/match relation after enclosing-return records are available,
  and `x86_node_inst_counts` consumes explicit parser/type/backend relation
  rows instead of binding tree parent/subtree buffers. Parent/subtree data is
  still present in dedicated same-end, order, and pointer-jump relation passes,
  and `x86_node_inst_gen` still consumes the GPU-produced subtree instruction
  bounds for branch/call target rows and write-placement validation. Those
  bounds are not source-text recognizers, but they are still a bounded control
  target/placement contract rather than a fully explicit paper-style edge
  relation for every lowering case. `x86_virtual_regalloc` also still has a
  bounded value-definition chunk loop. The x86 backend now publishes an explicit
  `x86_regalloc_pass_contract()` plus capacity-trace status counters marking
  that allocator `bounded`, `fail-closed`, and `blocked`, so performance
  artifacts must not infer an unbounded pass contract from successful local
  codegen. Struct- and array-literal local store ownership now fail closed
  unless the resolved let initializer is exactly that literal.
- The active x86 backend now materializes compact relocation records after
  text-offset generation and before encoding, and `x86_encode.slang` fails
  closed if `reloc_status` is not successful. Encoding now scatters instruction
  bytes from compact rows instead of binary-searching instruction offsets per
  output byte. `x86_reloc_patch.slang` is wired after encoding and before ELF
  layout; it consumes compact relocation rows directly and atomically scatters
  rel32 bytes into packed output words.
  The remaining relocation blocker is separate object/interface relocation
  records for package-scale linking, not the resident whole-ELF patch boundary.

## Highest-Priority Blockers

1. Define and enforce the first production language slice across parser-owned
   HIR, typed semantic records, backend lowering, stdlib imports, diagnostics,
   and acceptance tests.
2. Replace bounded semantic/codegen slices with generic record-driven lowering:
   monomorphization/specialization, trait dispatch, enum payload layout/match
   lowering, methods, aggregate operations, and native trap/runtime boundaries.
3. Turn descriptor/link/runtime contracts into real object/interface/relocation
   artifacts and final native executable linking.
4. Make packages production-real: descriptor-mode package/source-root
   preparation, quoted/import policy, artifact identities for real outputs, and
   GPU-owned module/package diagnostics.
5. Execute and publish local performance evidence for 5k/10k/20k checkpoints
   before making speed or scaling claims.
