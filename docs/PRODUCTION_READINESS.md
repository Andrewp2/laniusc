## Evidence Rules

- `bounded` means the current slice has behavior, record, artifact, diagnostic,
  or public-boundary evidence in checked-in tests.
- Parser-only, type-check-only, descriptor-only, or scaffold-only evidence is
  not executable production support.
- Production gates must be proved by public behavior, GPU-owned records,
  persisted artifacts, stable diagnostics, or measured local artifacts. Tests
  that read compiler/shader source or grep implementation text are not
  readiness evidence. The current automated source-inspection audit enforces that
  rule on Rust integration tests, including direct Rust product-source
  include/open/read/metadata/listing probes as well as command-based source
  inspection; `lib:laniusc` or `bin:*` evidence under `src/` is restricted to
  public-boundary, artifact-contract,
  execution-contract, or measurement-scaffold rows before it can count as
  readiness evidence.
- Behavior and property tests are the preferred readiness evidence. A promoted
  test must state the property it protects: accepted/rejected behavior,
  deterministic name/shape independence, artifact round trips, GPU record
  invariants, execution output, or stable fail-closed diagnostics. Exact helper
  names, private buffer names, shader/source snippets, command spellings, and
  other implementation markers are not substitutes for that property.
- Fail-closed GPU evidence means a bounded negative contract, not support. It
  must prove that unsupported source shapes, record rows, ABI/runtime calls, or
  link/measurement artifacts are rejected at an observable boundary without
  publishing a success artifact or relying on CPU semantic fallback.
- Performance and scaling inventory rows must keep paper numbers, no-run plans,
  local artifacts, pass-contract evidence, link artifacts, and Pareas comparison
  provenance separated. They must name the paper pass order, blocked paper-pass
  alignment, blocked pass-contract statuses, non-empty pass-contract and
  alignment blockers, workload-shape/generalization boundaries, and scaling
  blockers that still include both `paper_pass_alignment:blocked` and a
  top-level `pass_contracts:blocked` scaling blocker. Pass-contract debt must
  not appear only as nested paper-alignment context.
- The CPU may load files, schedule work, and write artifacts. It must not parse,
  rewrite, type-check, monomorphize, link, or patch program semantics as a
  production fallback.

- `alloc::allocator` now exposes `allocator_is_known_but_unbound()` plus
  per-extern `*_is_known_but_unbound()` allocator probes. This is source-level
  stdlib-root type-check evidence only; it does not make heap allocation
  executable or bind the allocator runtime service.

- The measurement scaffold now records the paper pass order from the checked-in
  GPU compilation papers separately from local Lanius evidence. The local
  scaffold can cite that order only as a baseline architecture contract; it
  cannot promote paper timing tables, the narrow current optimization-stage
  coverage, bounded/fail-closed pass contracts, or optional Pareas provenance
  artifacts into performance, scaling, or Pareas comparison claims.
- Optional Pareas comparison artifacts now carry `pareas_source_line_count` in
  the measurement summary. If the generated Pareas input is smaller than the
  claimed checkpoint, the summary marks `pareas_source:line_count` stale, so a
  5k/10k comparison cannot use a smaller Pareas source as local comparison
  evidence.
- Type checking has moved visible declarations, predicate obligation arguments,
  and method contract names/owners onto parser-owned HIR records rather than
  late consumer-side source discovery. Struct member and
  struct-literal field lookup now consumes parser/type metadata before the
  consumers run: the type-instance pass seeds/sorts/scatters a field-key table
  keyed by `(struct_node, field_name_id)`, parser passes publish nearest
  struct-literal statement context rows, and type checking maps those into
  literal-keyed context rows before field typing performs range queries. It
  still has bounded obligation windows, parser-local record derivation, and
  incomplete nested contextual type propagation for struct literals.
  Those are alpha/fail-closed limits, not production scalability evidence.
- Source-pack package/import/link control-plane records now enforce coarse
  library ordering plus canonical persisted ids/ranges for library
  dependencies, package-lock import graph dependencies and edges, job
  dependencies, job-batch dependencies, work-queue edges, artifact ranges, and
  link-batch inputs; inline link-batch rows must also carry at least one
  concrete artifact input. This is still scheduling and replay integrity, not
  semantic module resolution or real GPU linking. Compact library schedule pages
  also have to replay against the current schedule index partition, frontend
  range, codegen range, and link job before chunked schedule or link-leaf
  preparation can reuse them.
- Source-pack library partition validation now rejects empty or
  under-file-count source-byte summaries before those partitions can feed
  scheduling or hierarchical link planning, keeping source provenance attached
  to package input records instead of descriptor summaries.
- Package lockfile loading now rejects lockfile files that live inside package
  or stdlib source roots, matching the existing write-side guard. Lockfiles
  remain control-plane replay metadata, not source files uploaded for GPU
  module/import semantics.
- Package lockfile replay now validates persisted input, source-identity, and
  import-graph section shapes before live source-path or optional artifact
  checks, so malformed replay metadata is not masked by missing files.
- Link execution store input must carry the concrete sidecar records that the
  store is about to persist. Pre-paged interface/object/partial-link counts are
  replay metadata, not artifact evidence, and are rejected before a top-level
  link execution page can be written. Persisted link sidecar pages also reject
  impossible dense link slots before they can publish input artifact evidence.
- Package-lock import graph replay now also treats user-root precedence over the
  stdlib fallback as a persisted artifact invariant: a hand-edited user import
  edge cannot target a stdlib module when source identities already include the
  same module path from a package/user root.
- Import dependency evidence is the live or replayed source import edge matched
  against validated source identities; coarse library dependency rows only
  schedule work. User/package sources may depend on the stdlib fallback only
  when user/package roots do not provide that module path, but stdlib sources
  cannot depend back into user/package libraries.
- Package roots are not package-wide source inventories. Lockfile replay must
  reject persisted input/source-identity rows that are not reachable from the
  entry through validated import-graph edges, so unrelated root files cannot
  become package evidence through hand-edited metadata.
- Stdlib-only import-boundary tests belong in this same evidence lane: a stdlib
  source importing another stdlib module must resolve within `--stdlib-root`
  even when a package/user root exposes the same module path, and replay
  metadata must not turn that stdlib import into a package/user dependency.
- Package-lock replay reports stale import target movement before a generic
  import-count mismatch when the same source/import edge now resolves to a
  different source file. Persisted import-graph paths remain replay metadata,
  not semantic module identity.
- x86 lowering still has bounded record consumers that are not final
  paper-style joins: `x86_return_match_records` now materializes the direct
  return-statement/match relation after enclosing-return records are available,
  and `x86_node_inst_counts` consumes explicit parser/type/backend relation
  rows instead of binding tree parent/subtree buffers. Parent/subtree data is
  still present in dedicated same-end, order, and pointer-jump relation passes,
  and `x86_node_inst_gen` still consumes the GPU-produced subtree instruction
  bounds for branch/call target rows and write-placement validation. Those
  bounds are not source-text recognizers, but they are still a bounded control
  target/placement contract rather than a fully explicit paper-style edge
  relation for every lowering case. The raw loop audit now has zero
  paper-pass blockers, zero review-required rows, and zero high/medium-risk
  rows, but its stricter `audit-evidence-role` rows split the 71 scanned loops
  into `proof=21`, `blocker=0`, and `local-review=50`. The proof rows are
  primitive-shape-only no-run evidence; the local-review rows are bounded
  helpers that still need behavior-facing invariants or rewrites. The x86
  codegen subset has zero
  review-required rows and two fixed-bound rows:
  `x86_encode.slang` scatters up to `X86_MAX_ENCODED_INST_BYTES`, and
  `x86_virtual_regalloc.slang` walks `MAX_REGALLOC_ROWS_PER_CHUNK` while the
  allocator status remains `bounded`, `fail-closed`, and `blocked`. The x86
  backend now publishes an explicit `x86_regalloc_pass_contract()` plus
  capacity-trace status counters, so performance artifacts must not infer an
  unbounded pass contract from successful local codegen. The no-run shader loop
  audit currently has an empty paper-pass/review/source-sized-cap blocker queue,
  but that is only the old non-local rewrite queue. The new audit-evidence rows
  keep bounded-helper local-review rows visible, so a zero
  `shader_loop_audit_blocker` pointer cannot be mistaken for a claimable
  parallel-pass contract; the audit policy marker
  `zero-paper-pass-blocker-not-pass-contract-proof` makes that boundary explicit
  in summary-only artifacts. Source-sized symbolic-cap candidates are currently
  zero. The legacy whole-module WASM shader and the single-body WASM byte
  emitter are retired from the production surface; current WASM tests cover only
  stable fail-closed diagnostics. Source-pack, function-call, and module-scale
  execution coverage belongs to the x86 backend until WASM is rebuilt from
  record/count/prefix-sum/scatter passes.
  Struct- and array-literal local store ownership now fail closed unless the
  resolved let initializer is exactly that literal.
- The active x86 backend now materializes compact relocation records after
  text-offset generation and before encoding, and `x86_encode.slang` fails
  closed if `reloc_status` is not successful. Encoding now scatters instruction
  bytes from compact rows instead of binary-searching instruction offsets per
  output byte. `x86_reloc_patch.slang` is wired after encoding and before ELF
  layout; it consumes compact relocation rows directly and atomically scatters
  rel32 bytes into packed output words.
  The remaining relocation blocker is separate object/interface relocation
  records for package-scale linking, not the resident whole-ELF patch boundary.

## Paper-Alignment Gap List

Use this as the production-readiness checklist when comparing the repo to
`docs/CompilationOnTheGPU.md`, `docs/ParallelCodeGeneration.md`,
`docs/ParallelLLParsing.md`, and
`docs/ParallelLexingParsingSemanticAnalysis.md`:

- **Parallel parser:** adjacent-pair parser artifacts and seeded LL(1) block
  machinery are not enough. The readiness gate is a deterministic LLP/PSLS
  reduction that feeds resident HIR/type/codegen, with no LL(1) replay or
  witness-projection oracle in the production path.
- **AST/HIR materialization:** production cannot stop at a production-id tree
  plus HIR-facing classification buffers. Each supported construct needs stable
  parser-owned owner/operand/type records consumed by semantic and backend
  passes.
- **Semantic analysis:** bounded predicate, generic, method, and value-call
  bridges remain alpha evidence. The paper-aligned gate is compact relation
  publication with prefix scans, segmented scans for per-owner ranges,
  sort/join/scatter passes, and reduction validation, plus behavior tests for
  the supported slice and stable fail-closed diagnostics for any intentional
  cap.
- **Code generation:** x86 has resident record passes and relocation patching,
  and its audit-visible high/medium-risk and review-required codegen loops are
  reduced to zero, with bounded-local fixed loops still present in x86.
  The current optimizer is still narrow, and virtual register allocation remains
  a bounded/fail-closed pass contract. Readiness requires the paper ordering:
  node-local instruction counts, prefix-summed instruction locations,
  depth/layer-scheduled bottom-up virtual instruction generation with child
  results in fixed slots, then allocation/spill, byte sizing/scatter,
  relocation/fixup, and output. It also requires claimable
  `x86_regalloc_pass_contract()` status and supported direct-call,
  control-flow, aggregate, method, enum, and runtime boundaries to execute or
  fail with stable diagnostics.
- **Packages and linking:** the papers do not prove package builds. Lanius needs
  separate package libraries to emit real object/interface artifacts, GPU symbol
  and relocation rows, resumable link state, and final native output without CPU
  semantic rewriting.
  Completed reduce-link replay now rejects descriptor summaries that either
  overstate or silently drop interface/object/relocation records carried by
  consumed partial-link producer execution pages; final-page descriptor
  summaries are still metadata, not proof of GPU-owned
  interface/object/relocation rows or link-resolution artifacts.
- **Performance claims:** paper timings are reference-only. Readiness requires
  complete local `lanius.measurement-summary.v1` artifacts with timings,
  readback spans, VRAM/resource data, hardware/env hashes, source-control
  provenance, repeatability, a clean shader-loop-audit blocker signal, and
  claimable paper-pass/pass-contract fields. It also requires
  `link_artifact_evidence_schema=lanius.link-artifact-evidence.v1`,
  `link_artifact_required_evidence_classes=library_interface_artifacts,codegen_object_artifacts,partial_link_artifacts,linked_output_artifact`,
  and artifact-backed evidence for every named link artifact class.

## Highest-Priority Blockers

1. Define and enforce the first production language slice across parser-owned
   HIR, typed semantic records, backend lowering, stdlib imports, diagnostics,
   and acceptance tests.
2. Replace bounded semantic/codegen slices with generic record-driven lowering:
   monomorphization/specialization, trait dispatch, enum payload layout/match
   lowering, methods, aggregate operations, and native trap/runtime boundaries.
3. Turn descriptor/link/runtime contracts into real object/interface/relocation
   artifacts and final native executable linking.
4. Make packages production-real: descriptor-mode package/source-root
   preparation, quoted/import policy, artifact identities for real outputs, and
   GPU-owned module/package diagnostics.
5. Execute and publish local performance evidence for 5k/10k/20k checkpoints
   before making speed or scaling claims.
